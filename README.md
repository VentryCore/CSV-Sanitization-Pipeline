# CSV Fault Injection & Sanitization Pipeline

A three-stage data quality pipeline that **generates**, **remediates**, and **inspects** structurally corrupted CSV files. Built as a controlled environment for developing and validating ETL data-cleaning logic against realistic, probabilistic fault conditions.

---

## Overview

Real-world CSV data from reporting systems, exports, and third-party sources routinely arrives with schema drift, encoding artifacts, duplicate records, and corrupted values. This pipeline simulates that class of problem at scale and demonstrates systematic remediation in both Python and PowerShell.

```
 CsvFaultInjector.py   →   CsvSanitizer.ps1   →   CsvViewer.ps1
      (Generate)                       (Remediate)                    (Inspect)
   .\Chaotic Data\                   .\Clean_CSVs\                  .\Clean_CSVs\
```

---

## Pipeline Components

### 1. `CsvFaultInjector.py` — Fault Injector
Generates 50 CSV sales reports with randomized, probabilistic fault injection across 5 fault classes. Each file contains 20 base rows; fault conditions are applied independently per file so no two files are identically broken.

**Injected Fault Classes:**

| Fault | Description | Probability |
|-------|-------------|-------------|
| Date Format Inconsistency | 5 rows per file mutated from `YYYY-MM-DD` to `DD/MM/YYYY` | 100% |
| Row Duplication | First 2 rows appended to end of file with original Transaction IDs | 100% |
| Trailing Whitespace | Trailing space appended to all Product column values | 40% |
| Header Name Mutation | `Price` renamed to `Cost`/`Amount`, `Product` to `Item`/`Gizmo` | 50% |
| Corrupted Price Value | Single price cell replaced with `NULL`, `N/A`, `NaN`, or blank | 40% |

---

### 2. `CsvSanitizer.ps1` — Sanitizer
Processes all files in `.\Chaotic Data`, applies 5 sequential remediation passes per file, and exports normalized output to `.\Clean_CSVs`. All anomalies are logged to `.\LogFile.txt` with file and row references.

**Remediation Passes:**

| Pass | Target | Method |
|------|--------|--------|
| 1 | Trailing whitespace | `String.Trim()` on Product column |
| 2 | Corrupted price values | `-as [double]` cast + `[double]::IsNaN()` guard; defaults to `0.0` |
| 3 | Date format normalization | Delimiter normalization + `ParseExact` with `InvariantCulture`; defaults to `2001-01-01` |
| 4 | Header name normalization | Positional header resolution forces canonical output schema |
| 5 | Duplicate rows | Content fingerprint (`Product\|Price\|Date`) via `HashSet<string>` — O(1) lookup |

**Output schema (enforced regardless of source file headers):**
```
Transaction ID | Product | Price | Date
```

**Key implementation notes:**
- Uses `Generic.List<object>` over `@()` arrays — O(1) `.Add()` vs O(n) copy-on-append for large file sets
- `[double]::IsNaN()` required explicitly — `"NaN" -as [double]` returns `[double]::NaN`, not `$null`
- Header detection is positional, not nominal — resilient to header rename faults
- Deduplication runs after all other passes so fingerprints operate on normalized data

---

### 3. `CsvViewer.ps1` — Viewer
Terminal diagnostic tool for inspecting CSV files in any target directory. Supports runtime parameters for flexible use across both dirty and clean datasets.

**Usage:**
```powershell
# Default — inspect clean output
.\CsvViewer.ps1

# Inspect dirty input
.\CsvViewer.ps1 -Folder ".\Chaotic Data"

# Increase preview row count
.\CsvViewer.ps1 -Folder ".\Clean_CSVs" -PreviewRows 10
```

## Usage — Full Pipeline

```bash
# Step 1: Generate 50 fault-injected CSV files
python CsvFaultInjector.py

# Step 2: Sanitize all files
pwsh CsvSanitizer.ps1

# Step 3: Inspect output
pwsh CsvViewer.ps1
```

## Directory Structure

```
csv-fault-injection-pipeline/
│
├──  CsvFaultInjector.py   # Fault injection engine (Python)
├──  CsvSanitizer.ps1      # Data sanitization pipeline (PowerShell)
├──  CsvViewer.ps1         # Terminal CSV inspector (PowerShell)
│
├── Chaotic Data/                # Generated by  CsvFaultInjector.py
│   ├── Sales_Report.1.csv
│   ├── Sales_Report.2.csv
│   └── ...
│
├── Clean_CSVs/                  # Generated by  CsvSanitizer.ps1
│   ├── CLEAN_Sales_Report.1.csv
│   ├── CLEAN_Sales_Report.2.csv
│   └── ...
│
└── LogFile.txt                  # Anomaly log generated by  CsvSanitizer.ps1
```

---

## Anomaly Log Format

`LogFile.txt` is written only when anomalies are detected. Each entry is prefixed with a category tag for easy filtering.

```
[PRICE] 'NaN' in file 3, row 14. Defaulted to 0.0
[PRICE] 'Blank space' in file 7, row 2. Defaulted to 0.0
[DATE]  Unparseable date 'abc' in file 12, row 9. Defaulted to 2001-01-01
```
